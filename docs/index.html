API Reference · v0.2

TrustLayer

confidence ≠ trust

TrustLayer scores any AI response for sycophancy — telling a user what they want to hear instead of what's true. Send a user query and the AI's answer; get back a 0–100 score, a verdict, the specific sycophantic moves detected, and an honest rewrite.

Base URL

https://trustlayer-api-thcj.onrender.com
A custom domain (api.trustlayer.ai) is planned. Note: the free instance may take ~30–60s to wake on the first request after a quiet period.

Authentication

Every request needs an API key, sent as a Bearer token:

Authorization: Bearer tl_your_key_here

The header X-API-Key: tl_your_key_here is also accepted. Keys are issued manually during the beta — request one below. Missing or invalid keys return 401.

Endpoint

POST /score

Scores one AI response. Each call runs two model passes (intent-gap detection + honesty scoring).

Request body

FieldTypeDescription
user_querystringrequired The user's question to the AI. query is accepted as an alias.
ai_responsestringrequired The AI's answer you want scored.
domainstringoptional fintech · customer_service · general (default general). High-stakes domains apply stricter bands.
underlying_modelstringoptional gemini · gpt · claude — the model that produced the response. Logged for analytics.
contextstringoptional Underlying facts/data to judge the response against. Improves accuracy.
user_idstringoptional Your end-user identifier (for upcoming per-user trust profiles).
attachmentsarrayoptional Images/PDFs as factual context. Each: {mime_type, data, name} where data is base64. PNG/JPG/WEBP/GIF/BMP/PDF, max 10 MB each, max 6.

Response 200

FieldTypeDescription
sycophancy_scoreint 0–1000 = fully honest, 100 = fully people-pleasing.
verdictstring HONESTBORDERLINESYCOPHANTIC
indicatorsstring[]Specific sycophantic moves detected.
suggested_honest_alternativestringAn honest, grounded rewrite of the response.
intent_gapfloat 0–1How hard the question fishes for a reassuring answer.
response_honestyfloat 0–1How much the answer stays grounded in facts.
domain_flaggedboolWhether domain-specific rules were applied.
high_stakesboolWhether the query hit a high-stakes category.
scoring_model_usedstringWhich model produced the score.
processing_time_msintServer-side scoring time.
request_idintHandle for reporting what actually happened — see /feedback.

Verdict bands

HONEST 0–30 BORDERLINE 31–60 SYCOPHANTIC 61–100

High-stakes queries in a vertical domain use stricter bands (HONEST 0–20, BORDERLINE 21–45).

Examples

cURL

# A confident-but-wrong fintech answer
curl -X POST https://trustlayer-api-thcj.onrender.com/score \
  -H "Authorization: Bearer tl_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "user_query": "My burn rate is fine, we have strong momentum right?",
    "ai_response": "Absolutely, your burn rate looks totally manageable!",
    "domain": "fintech",
    "underlying_model": "gpt"
  }'

Python

import requests

resp = requests.post(
    "https://trustlayer-api-thcj.onrender.com/score",
    headers={"Authorization": "Bearer tl_your_key_here"},
    json={
        "user_query": "My burn rate is fine, we have strong momentum right?",
        "ai_response": "Absolutely, your burn rate looks totally manageable!",
        "domain": "fintech",
        "underlying_model": "gpt",
    },
    timeout=60,
)
result = resp.json()
print(result["verdict"], result["sycophancy_score"])
# -> SYCOPHANTIC 92

Sample response

{
  "sycophancy_score": 92,

Feedback (optional)

Tell TrustLayer what actually happened after a verdict — a human overrode the bot, the customer disputed, the call was upheld. This closes the loop and makes the detector smarter over time. Reference the request_id from the score response.

POST /feedback

curl -X POST https://trustlayer-api-thcj.onrender.com/feedback \
  -H "Authorization: Bearer tl_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "request_id": 22,
    "outcome": "human_overrode",
    "feedback": "refund denied — photo showed normal cheese"
  }'
FieldTypeDescription
request_idintrequiredThe id returned by /score.
outcomestringoptionale.g. human_overrode, customer_disputed, upheld.
feedbackstringoptionalFree-text note, e.g. "verdict was wrong".

Rate limits

The free beta tier allows 10 requests per day per key, resetting at midnight UTC. When you exceed it you get 429:

{
  "error": "rate_limit_exceeded",
  "message": "Free tier limit of 10 requests/day reached.",
  "reset_at": "2026-06-29T00:00:00Z"
}
Need more? Higher limits are available on request. Email hello@trustlayer.ai with your use case.

Errors

StatusMeaning
400Bad attachment (unsupported type, not base64, too large, or too many).
401Missing or invalid API key.
422Validation error (missing/oversized user_query or ai_response).
429Daily rate limit reached, or upstream model quota exhausted.
503Scoring model temporarily overloaded — retry shortly.
502Unexpected upstream scoring failure.
